แบบทดสอบ บทที่ 4
1. ให้แก้ข้อผิดพลาดใน Class Diagram ต่อไปนี้ให้ถูกต้อง (โดยยึดถือตามสิ่งที่ควรจะเป็นในของโลกความเป็นจริง)

a. ความสัมพันธ์ของ Class ที่มีส่วนเกี่ยวข้องกับงานแต่งงาน
โดย Bride หมายถึง เจ้าสาว และ Gloom หมายถึง เจ้าบ่าว
Guest หมายถึง แขกในงานเลี้ยงสมรส แยกออกเป็นญาติ (Relaties) ของคู่บ่าวสาวและเพื่อน (Friend) ของคู่บ่าวสาว
Party หมายถึง งานเลี้ยงสมรส


b. ความสัมพันธ์ของ Class ต่างๆ ที่เป็นองค์ประกอบของการชมการแสดง โดย Show หมายถึง การแสดง
Seat หมายถึง ที่นั่งชมการแสดง โดยตั่ว (Ticket) หนึ่งใบใช้สำหรับชมการแสดงหนึ่งที่ เท่านั้น ราคาของตั๋วจะแปรไปตามประเภทของที่นั่ง
Audience หมายถึง ผู้เข้าชมการแสดง (ผู้ซื้อตั๋วชมการแสดง)


c. ให้เพิ่มเติม Attributes และ Methods ที่ควรมีในแต่ละ Class ในข้อ a และ b

2. เขียน Class Diagram ที่สามารถอธิบายโครงสร้างและความสัมพันธ์ระหว่าง Class, Objects, Aggregation, Association และ Generalization (สามารถสร้าง Class หรือ Interface อื่นๆ เพิ่มเติมได้ หากจำเป็น) โดยไม่ต้องใส่รายละเอียดของ Attributes และ Method

3. สร้าง System Vocabulary จากรายละเอียดในแบบทดสอบข้อที่ 3 ของบทที่แล้ว (ข้อแนะนำ ควรเริ่มต้นการทำงานจาก Use Case Diagram ซึ่งเป็นผลลัพธ์ของแบบทดสอบข้อที่ 3 ของบทที่แล้ว)

4. สร้าง Business Class Diagram จาก System Vocabulary ที่ได้สร้างไว้แล้วในข้อที่ 3
