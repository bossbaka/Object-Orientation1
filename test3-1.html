แบบทดสอบ บทที่ 3
1. ให้สร้าง System Context Model และ System Requirement Model จากความสัมพันธ์ระหว่าง Actor กับ Use Case

2. จงตอบคำถามต่อไปนี้
    ก. Use Case “การถอนเงินเคาน์เตอร์ที่ธนาคาร” กับ Use Case “การตรวจสอบจำนวนเงินในบัญชีของผู้ถอนเงิน” ควรมีความสัมพันธ์แบบใด
    ข. Use Case “การสร้างบ้าน” กับ Use Case “การปรับเปลี่ยนแบบแปลนของบ้าน” ควรมีความสัมพันธ์แบบใด
    ค. Use Case “การซื้อตั๋วรถไฟฟ้าโดยซื้อจากเครื่องขายตั๋ว” กับ Use Case “การซื้อตั๋วรถไฟจากเจ้าหน้าที่” ควรมีความสัมพันธ์แบบใด
    ง. ให้ยกตัวอย่าง Main Flow of Events และ Exceptional Flow of Events (จำนวน 3 ตัว) ของ Use Case การถอนเงินในข้อ ก

3. หากเพิ่มเติมรายละเอียดของระบบการซื้อขายเงินตราต่างประเทศเป็นดังนี้ “การซื้อขายเงินตราต่างประเทศของสถาบันการเงิน (Foreign Currency Trading) จำแนกเป็น การซื้อขายเช็คเดินทาง (Traveller Cheque Trading) และการแลกเปลี่ยนเงินต่างสกุล (Foreign Exchange) ซึ่งการแลกเปลี่ยนเงินต่างสกุลนี้ จำแนกออกเป็น การแลกเปลี่ยน ณ เวลานั้น (Spot Foreign Exchange) และการแลกเปลี่ยนล่วงหน้า (Forward Foreign Exchange) โดยลูกค้าที่สามารถแลกเปลี่ยนเงินต่างสกุลแบบแลกเปลี่ยนล่วงหน้าได้นั้น ต้องเป็นลูกค้าที่มีบัญชีกับธนาคาร (Accounted Customer) ส่วนลูกค้าทั่วไป (Ganeral Customer) สามารถทำรายการซื้อขาย ฯ อื่น ๆ ได้ทั้งหมด อนึ่ง ลูกค้าที่มีบัญชีกับธนาคาร ก็สามารถทำรายการซื้อขาย ฯ อื่น ๆ ได้เช่นเดียวกัน ในการซื้อขาย ฯ แต่ละครั้งระบบต้องสามารถทำการคำนวณข้อมูลค่างเงินที่จะซื้อจะขายของแต่ละสกุลเงินได้ โดยพิจารณาอัตราแลกเปลี่ยน (Exchange Rate) ของวันนั้น ๆ อย่างไรก็ตาม รายการซื้อขายเงินตราต่างสกุลอาจไม่สามารถดำเนินการได้หากเงินบางสกุลที่ลูกค้าต้องการซื้อหรือแลกเปลี่ยนนั้น ไม่มีอยู่ในคลังของธนาคาร แต่สำหรับการซื้อขายเช็คจะไม่ประสบปัญหานี้ และทุก ๆ ประเภทรายการของการซื้อขายเงินตราต่างประเทศลูกค้าต้องจ่ายค่าธรรมเนียมในการทำธุรกรรมให้แก่ธนาคารเสมอ รายการแต่ละประเภทจะมีอัตราค่าธรรมเนียม และวิธีการคิดค่าธรรมเนียมที่แตกต่างออกกันไป” ให้สร้าง System Requirement Model จากรายละเอียดข้างต้น

4. หากมีการเปลี่ยนแปลงรายละเอียดของระบบ ดังต่อไปนี้
    - การรับรายการสั่งสินค้า สามารถรับรายการสั่งสินค้าจากโทรศัพท์หรือจากระบบอินเทอร์เน็ตก็ได้
    - เมื่อมีการสั่งสินค้า ลูกค้าต้องจ่ายค่าธรรมเนียมส่งสินค้าเสมอ
    - ลูกค้าสามารถคืนสินค้าได้ หากสินค้านั้นมีข้อบกพร่อง
    ให้ System Requirement Model จากรายละเอียดที่เปลี่ยนแปลงข้างต้น
